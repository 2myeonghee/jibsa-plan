SECTION 01
문서 개요 및 배경

작성 목적

본 문서는 법무법인 더에이치황해 아파트 자문팀이 아파트집사·집변 플랫폼 운영 과정에서 식별한 업무 병목·운영 리스크·이용자 이탈 요인을 해소하기 위한 17개 개발 개선 요청사항을 우선순위별로 정리한 통합 기획서입니다.

핵심 개선사항: 특히 즉시 착수 항목인 No.2 사실관계 요약 기능 개선안에 대해 현행 구조 분석·비교 및 개선 양식을 Section 04에서 상세히 제안합니다.

SECTION 02
우선순위 선정 기준

각 항목의 우선순위는 아래 3가지 기준의 업무 영향도를 종합하여 자문팀이 판단하였습니다.

①
워크플로우 병목 해소
사건 처리 파이프라인 병목 구간 해소. 수동 처리로 인한 누락·지연 발생 여부.
②
운영 리스크
기능 부재 시 사건 추적 불가, 담당자 혼선, 정보 오류 등 운영 사고 발생 여부.
③
이용자 이탈
이용자가 서비스 이용 중 불편·불안을 느껴 이탈할 수 있는 수준의 영향도.

우선순위별 전체 개발 요청사항 (17개 항목)
우선순위 No. 항목 적용 범위 요청 이유 및 세부 내용
🔴 즉시 착수 1 테스트 서버 계정 집변 이용자 자문팀이 직접 사용해봐야 나머지 요청의 검증·피드백 가능. 전체 개선의 전제조건.
🔴 즉시 착수 2 사실관계 요약 기능 집사 집변 어드민 의견서 작성 소요시간 단축의 핵심. 현재 수동 입력 과정을 어드민에서 자동화. → Section 04 상세 설계 참조
🟠 빠른 후속 3 담당직원 계정·권한 / 직원명 기재란 집사 집변 어드민 변호사-직원 간 의사소통에 필수. 건별 담당자 파악 필요. 계정 공유 운영 후 의견서 작성 시 담당직원 입력란 신설 또는 직원 직접 기재 방안 중 논의.
🟠 빠른 후속 4 의견서 발송 후 후속 관리 집사 집변 어드민 의견서 발송 완료 후 이력 관리 기능. (변예지 대리 생성물 참고)
🟠 빠른 후속 5 직원 초안 발송 → 변호사 검토 요청 + 알림 집사 집변 어드민 검토 워크플로우의 핵심. 수동 운영 시 사건 누락·지연 확정 발생. 이용자 직발송 기능과 별개로, 직원 → 변호사 검토 요청 전용 버튼 신설 필요.
🟠 빠른 후속 6 자동저장 기능 집사 집변 어드민 데이터 유실 방지. 의견서 작성 중 이탈·오류 시 작성 내용 자동 보존.
🟠 빠른 후속 7 이용자 / 관리자 상태값 집사 집변 어드민 내부 사건 진행 상태 파악. 접수·검토·완료·발송 등 단계별 상태값 정의 필요.
🟡 중기 개선 8 증거자료 파일 설명 입력 집사 이용자 첨부파일 업로드 시 이용자가 간략한 설명 입력 → AI가 파일명 자동 변환. (예: "KakaoTalk_20260505" → "김철수 이체내역 증빙")
🟡 중기 개선 9 집사 → 집변 이관 기능 집사 집변 어드민 이용자 재접수 없이 카테고리 변경. 개발팀 설계 가능 여부 논의 선행.
🟡 중기 개선 10 접수 시 이용자 지위·성명 표시 집사 집변 어드민 회원가입 연동 기술 검토 선행. (예: ○○아파트 ○단지 관리소장 홍길동)
🟡 중기 개선 11 상담 진척도 UI 집사 집변 이용자 현재 단계·잔여 질문 수 표시로 이탈 방지. 마지막 질문 처리 시 "최종 정리 생성 중입니다." 안내. (MBTI 진행도 방식 참고)
🟡 중기 개선 12 민감정보 입력 경고 팝업 집사 집변 이용자 계좌번호·전화번호 등 민감정보 감지 시 "민감 정보가 포함되었습니다. 그대로 전송할까요?" 경고 팝업.
🟢 후순위 13 파일 업로드 오류 로그 확인 집사 집변 어드민 파일 업로드 불가로 사무장 개별 접수 시, 해당 시점 오류 로그 기록 확인 기능.
🟢 후순위 14 AI 질문 5회 제한 집사 집변 이용자 이용자 피로도 절감. 최대 5회 질문 후 자동 정리 단계로 전환.
🟢 후순위 15 계약아파트 기능 활성화 집사 어드민 회원가입 연동 기술 검토 선행.
🟢 후순위 16 계약서 검토 시 출처 표기 집사 집변 어드민 품질 개선 차원. AI 모델 기술 범위 확인 선행.
🟢 후순위 17 증거분석 치트시트 도입 집사 집변 어드민 현재 문의 단계. 정식 개발 요청 전 방향 확정 필요.
